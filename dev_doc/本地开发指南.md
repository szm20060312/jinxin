# 《晋·信》本地开发指南

> 目标：本地环境与 EdgeOne Pages 生产环境差异最小化，所有数据存 localStorage，零后端依赖。

---

## 一、环境要求

| 工具 | 最低版本 | 验证命令 |
|------|---------|---------|
| Node.js | 18.0+ | `node -v` |
| npm | 9.0+ | `npm -v` |

> macOS 用户建议用 Homebrew 安装：`brew install node`

---

## 二、快速启动

```bash
# 1. 进入项目目录
cd "/Users/szm/vibe coding/晋·信"

# 2. 安装依赖（首次或 package.json 变更后执行）
npm install

# 3. 启动开发服务器
npm run dev
```

启动后浏览器自动打开 `http://localhost:3000`（若端口被占用自动递增到 3001/3002/…）。

---

## 三、可用命令一览

| 命令 | 用途 | 何时使用 |
|------|------|---------|
| `npm run dev` | 启动 Vite 开发服务器（HMR 热更新） | 日常开发 |
| `npm run build` | TypeScript 类型检查 + 生产构建 → `dist/` | 部署前验证 |
| `npm run preview` | 本地预览生产构建产物 | 构建后验证 |
| `npx tsc --noEmit` | 仅 TypeScript 类型检查（不构建） | 快速验证代码 |

## 五、环境配置说明

### 5.1 本地 vs EdgeOne 差异表

| 维度 | 本地开发 | EdgeOne Pages 生产 |
|------|---------|-------------------|
| 静态文件服务 | Vite dev server (HMR) | EdgeOne CDN |
| 端口 | 3000（自动递增） | 80 / 443 |
| SPA 回退 | Vite 内置 history fallback | `public/_redirects` 规则 |
| 数据持久化 | `localStorage`（浏览器） | 同左侧 |
| 热更新 | ✅ HMR 即时生效 | ❌ 需重新构建部署 |
| 构建产物 | 内存中 | `dist/` 目录 |
| 环境变量 | `.env`（空模板） | EdgeOne 控制台配置 |
| Source Map | 有 | 无（`sourcemap: false`） |

**核心原则**：本地与生产仅构建方式不同，运行时行为完全一致。所有状态存 `localStorage`，无需任何后端服务。

### 5.2 无需配置的环境变量

当前项目无外部 API 依赖，`.env` 仅作为未来扩展模板存在。后续接入 Supabase 时在此添加：

```env
# 示例（当前不需要）
# VITE_SUPABASE_URL=https://xxx.supabase.co
# VITE_SUPABASE_ANON_KEY=xxx
```

### 5.3 SPA 回退机制

- **本地**：Vite dev server 自动将未知路由回退到 `index.html`
- **生产**：`public/_redirects` 文件在部署时被 EdgeOne Pages 读取，规则 `/* → /index.html 200`

## 六、开发工作流

### 日常开发（推荐）
```bash
npm run dev
# 修改代码 → 浏览器自动刷新
# 每次保存前建议先运行 npx tsc --noEmit 检查类型
```

### 提交前验证
```bash
# 1. 类型检查
npx tsc --noEmit

# 2. 生产构建验证
npm run build

# 3. 预览生产构建
npm run preview
```

### 数据排查
```bash
# 在浏览器 DevTools Console 中执行：
localStorage.getItem('jinxin_users')        # 查看所有注册用户
localStorage.getItem('jinxin_current_user') # 查看当前登录用户
localStorage.key(0)                          # 首条数据
```

### 清除游戏数据
```bash
# 浏览器 Console：
localStorage.clear()  # 清除所有数据（包括账号）
```

## 七、常见问题

### Q1：端口 3000 被占用
Vite 会自动尝试 3001、3002……终端输出中查看实际端口。或手动指定：
```bash
npx vite --port 3005
```

### Q2：TypeScript 编译报错
```bash
npx tsc --noEmit
```
根据错误逐条修复。常见原因：未使用的导入/变量。

### Q3：构建后页面空白
检查浏览器 Console 是否有 JS 报错。最常见原因：
- 资源路径错误（应使用根路径 `/`）
- SPA 回退未配置

### Q4：Hot Reload 不生效
重启 dev server：`Ctrl+C` 终止后重新 `npm run dev`。

## 八、EdgeOne 部署准备清单

当准备好部署时，只需两步：

1. **构建**：`npm run build`（输出 `dist/` 目录）
2. **上传**：将 `dist/` 目录内容上传到 EdgeOne Pages，启用 SPA 模式

| 构建产物 | 大小 |
|---------|------|
| `index.html` | 0.5 KB |
| `assets/index-*.css` | 14.6 KB → 3.6 KB gzip |
| `assets/index-*.js` | 208 KB → 75 KB gzip |
| **总计** | ~223 KB → ~79 KB gzip |

`dist/` 目录即为 EdgeOne Pages 的站点根目录，无需额外配置。
