# 《晋·信》开发文档

## 上篇：远景愿景

> 上篇为游戏的完整愿景设计，包含世界观、经济系统、玩法蓝图。它是设计基调的锚点，也是远期迭代的参照。MVP 开发请直接阅读**下篇**。

一、背景故事
楔子：天下变局中的一纸契约
嘉庆十二年秋，白莲教之乱的余烬在川楚的深山里化作最后一缕青烟。紫禁城内，嘉庆帝对着空虚的国库长叹；而在千里之外的山西平遥，南大街的青石板路上，驼铃声与算盘声交织成另一曲帝国的脉搏。这里没有皇权的威严，却有着比皇权更实在的力量——白银的流动。
大清帝国由盛转衰，官府的驿道上奔波的已不单是八百里加急的军报，更多的是民间票号传递的汇票与信件。一个横跨欧亚大陆的贸易网络，正以山西商人的脚步为针，以白银为线，悄然缝合起从武夷山茶田到恰克图皮货集市的天堑鸿沟。这是一场无声的变革，而它的心脏，就在你脚下的这片青砖灰瓦之间。
你的身份：票号大掌柜
在这座"大清金融中心"平遥城内，日升昌、蔚泰厚等大号的红漆招牌在日光下晃得人眼晕，它们的分号已如血管般伸向九省通衢。而你，执掌着一家中型票号。
它或许还没有那些老字号的赫赫威名，但有着自己的筋骨与傲气。东家是平遥本地大族，因厌倦了官场倾轧，将万贯家财全权委托于你——你是大掌柜，是这艘金融巨舟的实际掌舵人。总号之内，你执掌银库、调度分号、周旋官商、培养伙友。你既拥有独断的经营之权，也承担着泰山压顶般的经营之险。
那块悬挂在南大街门脸上的金字招牌，上面的字号由你亲手题写——它不是一个简单的名字，而是你将要倾注一生心血去守护的誓言。
你卖的是一张纸，守的是一口气
票号不卖米面绸缎，它卖的是一张薄薄的纸——汇票。这张纸之所以能让商人从京师提走汉口分号的万两白银，凭的完全是票号招牌的信誉。晋商经营数百年，立下了三条一旦触碰便形同自戕的铁律，而这，正是你每日面对的核心矛盾：
* 信为命脉：宁可赔光地窖里的存银，不可让任何一张盖上你票号印鉴的汇票无法兑付。哪怕遭逢天灾、劫匪或战乱，分号门前只要还有人手持真票，你就必须兑出白花花的银子。失信一次，便是灭门之灾。
* 利在长远：不做一锤子买卖。与相熟的商帮世代联姻，人情积累的厚度远胜于一时的高利。当有伙友兴奋地告诉你有一条"一本万利"但可能断人活路的路子时，你必须想起，晋商的生意，是做给子孙后代的。
* 庙堂与江湖的平衡：票号离不开官府，京师分号的存银有三成是暂存的库款；但你绝不能沦为官府的附庸。与巡抚大人走得太近，可能会让你在一夜之间获得全省的汇兑权，但也可能在他倒台抄家时，让你的票号连同你一起被拖入万劫不复的深渊。疏远则路难行，亲近则身易裹，这根钢丝，你必须悬着心走。
暗流之下的决断
此刻，你坐在票号后院昏暗的账房内，一盏油灯照亮了几份摊开的密函：
汉口分号来信，今年茶市受英吉利商船迟来之累，银根骤然紧缩，急需总号拨银二十万两救急。
张家口分号的老掌柜则附耳密报，恰克图的俄商对晋商汇票的信任日增，有意用大批毛皮换取汇兑服务，但对方要求派"通事"常驻，暗中窥探虚实，稍有不慎便会泄了密押底牌。
而平遥县衙的师爷方才登门，暗示新任巡抚欲将明年一省的地丁钱粮交由可靠票号汇兑，但这笔天大的买卖，需要你在账簿上"略作修饰"，为巡抚大人填补历年的积欠窟窿。
与此同时，你还要盯着前院那些新招的年轻伙友。他们必须学会用极复杂的暗码写汇票，必须练就闭眼一摸便知银两成色的绝活。在他们之中，或许藏着未来的分号掌柜，也可能潜伏着竞争对手派来的探子，或是某个前朝遗贵之后。
你的每一次落笔，每一次点头或摇头，都是在信义与利益、规矩与人情、商道与官道之间，寻找那一线细若游丝的平衡。
终极之路：汇通天下
东家将全部身家连同家族的荣辱都托付给了你。地窖里银箱的每一个角，分号里账本的每一页，都浸透着你的抉择。
你的目标绝非偏安一隅。你要让自己的票号翻过太行山，渡过长江，跨过戈壁，让拿着你票号票据的商人在帝国的任何一个角落都能立兑白银。你要从夹缝中求生存，在乱世里筑起一座不倒的金融堡垒。
当未来的史书翻开这一页时，那块由你亲手题写的金字招牌，将比任何白银都更加闪耀。
这，就是你的征程。从这间青砖灰瓦的账房开始，你的商道、你的信义、你的传奇，正等待被你亲手书写。
字号尚未落笔，传奇由你命名。掌柜的，请题写招牌！

二、核心资源体系：游戏的经济系统建立在三大核心资源之上。它们彼此独立又深度联动，构成票号经营的根本。
1. 白银 —— 流通的血液
定义：票号持有的实物白银总量，是所有业务的最终兑付基础。
细分结构：
层级	名称	作用
基础层	库银	各分号地窖中的实物白银，用于日常兑付
调度层	在途银	正在运输途中的白银，无法动用且有风险
缓冲层	存银	总号地窖中的战略储备，应对挤兑与突发
核心机制：
* 准备金率：库银必须维持在总存款的一定比例以上。低于此线，触发信誉危机。
* 异地调拨成本：调拨白银需耗费时间、运费和护卫成本，且途中有被劫风险。
* 白银与铜钱的兑换：基层百姓用铜钱，官商用白银。兑换比价随市场波动，票号可从中套利。
收支来源：
* 收入：汇费（汇兑手续费）、存银利息差、放款利息、官银汇兑手续费
* 支出：伙友薪俸、分号租金、护卫费用、运输成本、兑付支出

2. 信誉 —— 票号的灵魂
定义：票号招牌在商人、官府、百姓心中的信用程度。这是一张"纸"（汇票）能当银子用的根本。
细分维度：


维度	含义	影响因素
兑付信誉	能否随时足额兑付	每笔成功兑付+；拒兑/拖延-
密押信誉	防伪体系是否严密	密押被破译-；主动升级防伪+
江湖口碑	商帮间的人情评价	帮衬相熟商号+；落井下石-
官府印象	官场对你的态度	协办官银汇兑+；涉贪案牵连-
核心机制：
* 信誉的"易毁难建"法则：百年积累可毁于一夕。一次拒兑或密押泄露，信誉断崖式下跌。
* 信誉的连锁效应：某地分号出问题，会波及其他分号的存款意愿。
* 信誉兑换额度：信誉等级决定你能吸纳的存款上限、能承接的官银汇兑规模。

3. 人脉 —— 看不见的网
定义：大掌柜及其票号在商界、官场、民间编织的关系网络。
细分维度：


类型	来源	作用
商帮交情	与徽商、浙商、粤商等商帮的往来	获取商业情报、大宗汇兑业务
官府人脉	与各级衙门的关系	承接官银汇兑、获取政策信息、寻求庇护
东家信任	东家对你的满意度	决定放权程度、追加资本与否
乡党情谊	平遥本地士绅、同业关系	危机时拆借银两、伙友招募渠道
核心机制：
* 人脉的维护成本：逢年过节送礼、宴请、帮衬对方生意，都需要银两和精力。
* 人脉的双刃剑：与某官员过密，他升迁你受益，他倒台你遭殃。
* 人脉的传承性：培养伙友也是在传承人脉。老掌柜退休，人脉可部分转移给继任者。

三、机构与人力：经营的骨架
1. 分号网络
概念：你的票号在全国各地设立的分支机构。
等级体系：
* 总号（平遥）：决策中枢、银库总部、密押中心
* 一等分号（省会级）：汉口、苏州、广州、西安等，存银量大，业务种类全
* 二等分号（府城级）：张家口、恰克图、库伦等，服务于特定贸易路线
* 代办处（县镇级）：由当地商号代理，成本低但风险高
扩张逻辑：
* 新设分号需要：筹备银两（启动资本）、派驻掌柜（人力）、打通当地官商人脉
* 分号越多，汇兑网络越广，业务量越大，但管理成本和风险也同步上升
* 分号之间需要"联号制度"——定期对账、巡查，防止分号掌柜做假账、挪用库银

2. 人力资源
概念：票号所有人员，从学徒到大掌柜。
岗位体系：


岗位	职责	培养周期	风险
学徒（小伙计）	杂役、练字、练算盘、学密押	3-5年	不堪造就、被对手挖走
跑街（业务员）	外出揽储、收集商情	持续锻炼	私吞汇款、传递假信息
账房（会计）	记账、核算、防伪验证	长期积累	做假账、泄密
分号掌柜	执掌一地分号	需10年以上资历	独断专行、携款潜逃
大掌柜（玩家）	执掌全局	—	被东家撤换、被官府查办
核心机制：
* 培养与选拔：伙友从学徒做起，通过"修身、习业、历练"三个阶段成长。需投入时间、银两（薪俸、教导）和人脉（带出去历练）。
* 忠诚度与牵制：委任分号掌柜时，需要留其家眷在平遥作为"人质"（行业惯例），否则背叛风险大增。
* 特殊人才：有极低概率招募到天赋异禀的伙友（过目不忘、算盘快手、擅长交际），培养得当可成为左膀右臂。

四、业务系统：银两的流转之道
1. 汇兑业务（核心）
运作流程：商人甲在A地将白银交给你 → 你开具汇票 → 商人甲（或其代理人）持票在B地分号兑出白银 → 你收取汇费。
利润来源：汇费（手续费，通常为汇兑额的2%-5%）
风险控制点：
* 密押防伪：每张汇票需暗藏多重防伪（专用纸张、水印、笔迹暗记、密押编码），防伪等级决定被伪造的概率。
* 空头汇票：A地分号掌柜与商人合谋，开出没有实银支撑的汇票在B地兑付，导致B地库银亏空。需靠巡查和联号对账防范。
* 路途风险：在途调拨白银可能遇劫。需雇佣镖局或与军方合作。

2. 存银业务（基础）
运作：商人将闲置白银存入票号，票号出具存银票，到期支付本息。
利润来源：
* 存银利息差（存款利息低，放款利息高）
* 利用存款进行资金周转，承接更多汇兑业务
风险：
* 若存款集中到期，而库银被挪用于放款或汇兑，则面临挤兑危机。
* 需合理安排存期结构，避免"短存长贷"的错配。

3. 放款业务（扩展）
类型：
* 短期拆借：借给急需用钱的商人，期限短、利息高、风险大
* 长期贷款：借给商帮用于茶叶、丝绸等大宗贸易，期限长、利息稳定
* 官府垫款：为官府垫付军饷、赈灾款，利息高但有被赖账的可能
利润来源：利息收入
风险控制：
* 需要抵押品（货单、地产、古董）或担保人（有声望的商号作保）
* 需评估借款方的还款能力和品行（通过人脉网络收集情报）
* 坏账率随经济形势波动。天灾、战乱、洋货冲击都可能导致大面积坏账

4. 官银汇兑（战略级业务）
概念：各省上解中央的赋税、中央拨付地方的军饷赈灾款，委托票号代为汇兑。
吸引力：汇兑额巨大，汇费丰厚，且能极大提升信誉和官府人脉。
风险：
* 政治风险：押送官银的官员若被查办，票号可能被牵连，扣上"伙同贪墨"的罪名。
* 账目风险：官府要求账簿上"略作修饰"，若答应则违法，不答应则失去业务。
* 拖延付款：朝廷财政紧张时，会拖欠或赖掉汇费。

五、情报与风险控制系统
1. 商业情报网
信息来源：
* 各分号跑街的日常报告
* 相熟商帮的口头传递
* 官府师爷、衙役的私下透露
* 竞争对手的公开动向
情报类型：
* 商机类：某地茶价暴涨、俄商急需皮毛
* 风险类：某地即将战乱、某商帮资金链断裂
* 竞争类：某票号在何处新设分号、某票号密押被破译

2. 风险预警指标


指标	警戒线	危险后果
准备金率	< 30%	信誉开始下滑
单一分号存银占比	> 总存银40%	该分号出问题即全局危机
官银业务依赖度	> 总业务50%	官场震荡则票号必死
掌柜异动率	频繁离职	可能有内奸或管理危机
坏账率	> 10%	流动资金枯竭

六、完整玩法设计（远景）

基于之前构建的经济系统，下面是《晋·信》的完整玩法设计方案，按模块分类，注重策略深度与游戏性的平衡。

---

(一) 核心经营循环：大掌柜的日常

这是玩家在每个经营周期中进行的主要操作，构成游戏的基础循环。

1. 晨会决策（每日/每回合起始）

形式：每天清晨，账房先生将当日的紧要事项呈报在你案头。

玩法：
- 银库日报：查看各分号前一日存银变动、在途白银位置、准备金率。需在若干紧急事项中做出批示。
- 汇票审批：当日需要加盖总号印鉴的大额汇票（如超过五万两）逐张呈上，玩家必须亲自核对密押无误后方可用印。
- 情报简报：跑街收集的各地商情、官府动向、同业传闻，由账房摘录呈报。玩家可标记重点关注事项。

游戏性：信息不对称。你看到的都是经过筛选的信息，分号掌柜可能报喜不报忧。过度事必躬亲会拖慢效率（每日可处理事项有限），过于放手则可能埋下隐患。

---

2. 票号经营（季度性重大决策）

每个季度末，你需要与账房和主要掌柜议事，做出以下决策：

业务调度
决策项	内容	战略意义
资金调拨	决定各分号之间的白银调拨额度与路线	平衡各地准备金率，防范挤兑
汇费定价	调整不同路线的汇兑手续费	与同业竞争，薄利多销还是高利少单
存银利率	调整吸纳存款的利率	利率高则存银多但利差薄，反之则存银少
放款额度	决定本季度放款总量与单笔上限	扩大利润与风险控制的平衡

人事调度
决策项	内容	战略意义
掌柜轮调	将某分号掌柜调往他处	防止一地坐大，或在危机时分号需要能人
伙友提拔	从学徒中选拔跑街，从跑街中提拔账房	人才梯队建设
特殊派遣	派遣心腹巡查某分号账目	发现贪腐或假账

游戏性：每个决策都有延迟反馈。本季度调拨的白银，可能三个月后才到；提拔的伙友，可能一年后才看出是忠是奸。

---

3. 年终决算（年度大事件）

形式：年终腊月，各分号掌柜携账本返回平遥总号，进行一年一度的"合账"。

玩法：
- 对账小游戏：各分号账目汇总，与总号底账比对。若出现"对不上"的情况（系统预设的异常），你需要判断是记账失误还是有人做假账。可以亲自查账（耗费时间），也可以轻信解释（风险自担）。
- 分红与留利：决定本年利润中多少分给东家（影响东家信任）、多少留作票号资本（影响发展速度）、多少分给伙友作为花红（影响士气）。
- 人事任免：根据各分号掌柜一年表现，决定续任、升迁、调离或辞退。
- 新年规划：制定来年目标，如新设若干分号、承接某省官银汇兑等。

游戏性：合账是发现内部问题的最佳时机，但也最考验判断力。放过一个做假账的掌柜，他会变本加厉；冤枉一个忠心的掌柜，会寒了众人的心。

---

(二) 分号经营：帝国版图的扩张

1. 设立分号

前提条件：
- 白银储备达到门槛（视分号等级而定）
- 有合适的分号掌柜人选（资历、忠诚度达标）
- 在目标城市有一定人脉基础（或可派人事先打点）

流程：
1. 选定城市（提供候选城市列表，各有不同的商业潜力与风险评级）
2. 拨付筹备银两（决定分号初始存银规模，影响抗风险能力）
3. 任命分号掌柜（人选的能力与忠诚度决定分号经营质量）
4. 派遣随行伙友（账房、跑街等，影响分号起步效率）
5. 筹备期（3-6个月），期间有随机事件：当地官绅态度、同业竞争反应等

游戏性：分号不是越多越好。每增加一个分号，管理的复杂度呈指数上升，你可能面临"鞭长莫及"的困境。

---

2. 分号日常管理

每个分号有独立的状态面板：
指标	含义	影响因素
库银	当前存银量	汇兑流入流出、存款增减
准备金率	库银/应付存款	过低则触发信誉警报
业务量	每月汇兑笔数与金额	当地商业活跃度、票号信誉、汇费定价
掌柜状态	忠诚度、贪腐倾向、工作状态	监督力度、待遇、个人品行

玩法：
- 定期收到分号掌柜的汇报信（内容由系统根据分号状况和掌柜品行生成，可能有隐瞒或粉饰）
- 可派出巡查（耗费成本，但能发现真实情况）
- 可设置"密报"制度：在分号中安插直接向总号汇报的眼线（影响分号内部信任）

游戏性：分号掌柜是独立AI，有自己的性格和利益取向。忠厚者如实汇报但可能经营保守；精明者能赚大钱但也可能动贪念。如何用人和制衡，是贯穿游戏的核心挑战。

---

3. 危机事件（分号层级）

每个分号可能触发以下危机：
危机类型	触发条件	表现	应对选项
挤兑风潮	当地谣言、同业倒闭连锁反应	存银者大量提款，库银骤降	从总号急调白银 / 向同业拆借 / 暂停兑付（严重损害信誉）
密押泄露	内奸、被盗窃、破译	出现伪造汇票，分号未察觉而兑付	全系统更换密押（成本巨大）/ 追查内奸 / 报官
在途被劫	路遇匪患	调拨的白银或汇票被劫	雇佣镖局追讨 / 报官 / 自行承担损失
掌柜叛逃	掌柜忠诚度极低且有巨额银两经手	携款潜逃	报官通缉 / 动用江湖人脉追捕 / 向东家请罪
水火天灾	随机事件	分号库房被淹、被烧	重建分号 / 转移业务至邻近分号

游戏性：每次危机的应对都有多重路径，不同选择消耗不同的资源（银两、信誉、人脉、时间），且产生连锁效应。公开报官可能引来官府过度关注，私下解决则可能欠下江湖人情。

---

(三) 人才培养：从学徒到大掌柜

1. 伙友招募

途径：
- 乡党举荐：平遥本地士绅推荐同乡子弟，知根知底但素质参差
- 公开招考：面向社会招聘，可筛选素质但可能招入探子
- 商帮引介：合作商帮推荐的子弟，自带人脉但可能有利益牵绊
- 捡漏奇才：极小概率随机事件，遇到天赋异禀的年轻人

招募界面：每位候选伙友展示以下属性：
- 资质：天资聪颖 / 中人之姿 / 愚钝但勤恳
- 品行：忠厚老实 / 机灵圆滑 / 心术不正（隐藏属性，需长期观察）
- 出身：农家子弟 / 商贾之后 / 落魄书生 / 不详
- 特殊天赋：过目不忘、铁算盘、巧舌如簧、胆大心细等

---

2. 培养路径

伙友入职后，进入培养阶段：
阶段	时间	内容	玩家可操作项
学徒期	1-3年	杂役、练字、背密押、打算盘	指定师傅、观察品行
跑街期	2-5年	外出揽业务、收集情报	分配跟随的资深跑街、指定跑动区域
账房期	3-5年	学习记账、核账、防伪	接触部分核心密押
候选掌柜	不定	已具备执掌分号的资格	决定是否外放

玩家决策点：
- 是否让某个伙友提前接触核心密押（加速培养但增加泄密风险）
- 是否将伙友外放到艰苦但能锻炼人的分号
- 如何平衡培养速度与忠诚度考验

游戏性：你亲手培养的伙友会对你产生"师承情谊"，忠诚度更高。但同时，你的对手也可能在挖你的人才。每个伙友的成长过程中都有若干次"考验事件"——比如捡到客商遗落的银票、被竞争对手秘密接触等，这些事件的选择会塑造他的品行。

---

3. 师徒传承

机制：你可以指定资深伙友担任新学徒的"师傅"。

效果：
- 师傅的品行会潜移默化影响徒弟
- 师傅忠诚则徒弟大概率忠诚；师傅贪腐则可能师徒合谋
- 可形成"师门"派系，好处是凝聚力和忠诚链，风险是尾大不掉

游戏性：这是一种高风险高回报的人才策略。师徒相承可以大幅提升培养效率，但也可能在票号内部形成你难以掌控的利益小团体。

---

(四) 汇兑业务操作：核心玩法深度展开

1. 汇票开具流程

当客户（商人、官府）前来办理汇兑时，玩家（或由系统代理的分号掌柜）需完成以下步骤：

第一步：验银
- 客户交付白银，需当场验成色、称重量
- 小游戏：目测银锭成色（系统提供若干银锭的视觉差异），判断是否需要"折色"（成色不足则按比例折算）
- 一旦收下即视为认可，后续发现成色问题由票号承担

第二步：开票
- 填写汇票：金额（大写）、汇往地、取款人姓名或暗号
- 施加防伪：从密押库中选择本次使用的密押编码，暗记在汇票特定位置
- 加盖印鉴：票号公章、掌柜私章
- 收取汇费（现金或记账）

第三步：发信
- 同时向汇往地分号发出"对票信"（内部通知），告知即将有人持票兑付及密押信息
- 对票信需加密，走专门的传信渠道

---

2. 汇票兑付流程

第一步：验票
- 持票人来到汇往地分号，出示汇票
- 小游戏：在放大镜下检查密押暗记、笔迹特征、纸张纹理
- 若发现疑点，需决定：拒兑（可能得罪真客户）、照兑（可能被假票欺骗）、暂扣汇票请示总号（折中方案但拖延时间）

第二步：兑银
- 验票无误后，按票面金额兑付白银
- 同时在票面加盖"已兑付"印，将汇票存档备查

游戏性：验票环节是防伪的第一线，也是风险最高的环节。你可以为不同金额设定不同的验票权限——小额汇票由分号自行验票，大额汇票需总号确认。但这也意味着，大额汇款速度变慢，可能流失客户。

---

3. 密押系统（核心策略层）

机制：密押是票号的核心商业机密，是一套用于汇票防伪的编码体系。

密押设计：玩家可以自行设计密押的复杂度。游戏提供基础模板（如用特定字代表金额、用特定偏旁代表月份），玩家可在此基础上自定义。
- 密押越复杂，防伪越强，但伙友学习成本高，开票效率低
- 密押需定期更换（成本：全系统通知、伙友重新记忆）

密押泄露事件链：
1. 某分号出现假票兑付成功 → 密押可能部分泄露
2. 玩家需判断泄露范围和泄露渠道（内奸？被偷？被破译？）
3. 应对：局部更换密押（成本低但风险未根除）还是全面更换（成本极高但彻底）

游戏性：密押是"看不见的防线"。你永远不知道它是否已经被攻破，直到出现假票。这种不确定性本身就是一种持续的紧张感。

---

(五) 存银与放款：银行的本质

1. 存款吸纳

存款来源：
- 商人周转金（数额大、存期短）
- 官绅私蓄（数额大、存期长、对信誉敏感）
- 百姓零散积蓄（数额小但稳定）

玩家决策：
- 针对不同客群设置存款利率
- 是否提供"异地存取"服务（用汇兑网络为存户提供便利，吸引更多存款）
- 大额存款是否要求"暗记"以备查（加强安全性但让客户觉得不信任）

---

2. 放款审批

放款对象：
- 熟悉的商帮：安全度高，利息较低
- 陌生商人：需抵押或担保，利息较高
- 官府：利息高但风险特殊

审批流程：
1. 跑街收集借款方情报（经营状况、品行、担保人背景）
2. 掌柜或你本人判断：借款方还款能力、抵押品价值、担保人可靠性
3. 决定放款金额、期限、利息
4. 立借据、收抵押品

游戏性：情报永远不完整。你可能因为信息不足而错失好生意，也可能因为轻信而被坑。担保人制度引入了人脉资源的消耗——请相熟商号担保，是在消耗你与他们的人情。

---

3. 坏账追讨

当借款到期不还：
- 协商延期：给对方宽限，维持关系但占用资金
- 执行抵押：没收抵押品变卖（可能引发江湖恩怨）
- 找担保人：向担保人施压代偿（消耗人脉）
- 报官：走官府途径（官府抽成、效率不确定）
- 放弃追讨：认栽，记入坏账（影响账目但可能赢得对方感激）

游戏性：追债方式反映你的经营风格。铁腕追讨能减少损失但可能树敌；宽厚待人可能赢得长远友谊但也可能被视为软弱。

---

(六) 官府关系：走钢丝的艺术

1. 官银汇兑业务

获取途径：
- 主动竞标：多个票号争夺某省官银汇兑权
- 官员主动上门：对方有求于你
- 人脉引介：通过已有的官场关系拓展

竞标小游戏：
- 各票号报价（汇费率）、展示资质（信誉、分号网络）
- 官员也有私下考量：谁给的"回扣"多、谁的背景更可靠
- 玩家需在明面报价和私下运作之间分配资源

承接后的管理：
- 官银需单独建账，与商款严格分开
- 官员可能要求：提前支取未到账的款项、在账目上做手脚、为私人提供低息贷款
- 每次此类要求都是一次道德与利益的选择

---

2. 官场经营

人脉投资：
- 红白喜事送礼（银两开销）
- 官员调动时为其汇兑家产（业务机会）
- 在官员落难时是否伸出援手（高风险高回报）

政治风险：
- 你结交的官员可能因贪腐被查，票号账簿可能被牵连调查
- 地方官场派系斗争，你难以置身事外
- 应对抄家风险：预设"防火墙"（将官银业务与商款完全隔离）、准备"舍车保帅"（牺牲一个分号保全总号）

游戏性：与官府交往如同走钢丝。太近则危如累卵，太远则难成大事。需要你在每一次官场互动中，都留着后路。

---

(七) 江湖与人情：看不见的交易

1. 商帮关系

主要商帮：
- 晋商：你的本家，竞争与合作并存
- 徽商：主要经营盐、茶，富甲一方
- 浙商：经营丝绸、海外贸易，思想开放
- 粤商：广州十三行背景，与洋人有往来
- 陕商：西北贸易路线的主导者

关系维护：
- 互相帮衬生意（你有闲置白银时拆借给对方，反之亦然）
- 逢年过节互相拜会（需消耗时间和银两）
- 联姻（将伙友或亲属与对方家族联姻，长期绑定）

关系的作用：
- 好关系可获更好的汇兑价格、更及时的情报、危机时的支援
- 关系破裂则可能遭遇对方集体抵制，甚至联合其他票号围堵

---

2. 同业竞争

竞争形式：
- 价格竞争：降低汇费争夺客户（伤敌一千自损八百）
- 服务竞争：增设分号、提升密押安全性、提高兑付速度
- 人才挖角：高薪挖走对方骨干伙友
- 舆论攻防：散布对方信誉危机（可能反噬己身）

合作可能：
- 在挤兑危机中互相拆借白银（今日你帮我，明日我帮你）
- 联合承接超大型官银汇兑
- 共同制定行业汇费标准（类似价格同盟）

游戏性：同业关系在竞争与合作之间动态变化。你需要判断哪些对手是必须击败的，哪些是可以共存的，哪些关键时刻可能救你一把。

---

(八) 时代事件与历史进程

1. 历史背景事件（不可控大环境）

这些是系统根据时代背景定时触发的大事件，会对经营环境产生深远影响：
事件类型	示例	影响
经济波动	某年茶市暴跌、恰克图贸易中断	相关分号业务量骤减，放款坏账率上升
自然灾害	黄河决口、旱灾	灾区百姓存银被取出度荒，需考虑赈灾捐献
政策变动	朝廷整顿票号、限制白银外流	官银业务收紧、跨省汇兑需额外审批
中外关系	鸦片走私泛滥、白银大量外流	白银贬值，存户提银兑黄金，挤兑风险上升
白莲教余波	某地再次出现教乱	当地分号面临战乱风险，需紧急转移库银
皇帝驾崩/新皇登基	国丧期间商业活动受限	大额汇兑暂停，但新皇可能推行新政

游戏性：这些事件不可预测但可以应对。提前布局分散风险（不在同一区域过度集中分号）、预留应急白银、维护官商人脉以提前获取风声。

---

2. 特殊剧情事件

精心设计的固定剧本事件，在满足条件时触发：

示例一："密押之夜"
- 触发：总号大掌柜卧室内密押总册不翼而飞
- 进展：封锁总号、逐一排查、追溯近日可疑人员
- 分支结局：找回了（虚惊一场，加强安防）/ 确认泄露（全系统紧急换押，花费巨大）/ 发现是内奸所为（内部肃清，影响士气）/ 其实是对手虚张声势（情报战）

示例二："孤城分号"
- 触发：某分号所在城市突然爆发民变，城池被围
- 进展：分号与外界联系中断，库银无法转移，存户排队挤兑
- 你远程决策：信任分号掌柜自行处置（结果取决于该掌柜能力与忠诚度）、想办法打通关节运送白银进去（成本极高）

示例三："东家的考验"
- 触发：东家突然提出要亲自查账
- 可能原因：听信谗言、对利润不满、想收回经营权
- 应对：坦荡配合（需要账目干净）、暗中运作让查账不了了之（需人脉和银两）、主动请辞（赌东家挽留）

---

(九) 传承与终局

1. 大掌柜的更替

游戏中最具分量的时刻：你扮演的大掌柜终有年迈退休的一天。

传承决策：
- 从现有的分号掌柜或总号核心伙友中选择继任者
- 继任者继承你的人脉、信誉（按一定比例折损）
- 继任者的能力和品行，决定了你一生经营的事业是发扬光大还是迅速衰败

传承后：你将以"老东家"或"顾问"视角继续旁观，或者直接扮演继任者继续经营。

---

2. 终局判定

游戏没有固定的"胜利"终点，但有多种结局评价：
结局类型	条件	评价
汇通天下	分号覆盖全国主要商埠，信誉达到顶峰	一代传奇，青史留名
金玉满堂	白银积累达到惊人规模，富可敌国	商界巨擘，名利双收
守成之主	维持原有规模，平安度过一生	无过便是功，平安是福
晚节不保	晚年出现重大信誉危机或官非	百年基业毁于一旦
急流勇退	在巅峰期主动交班，安然退隐	大智慧，善始善终
惨淡收场	破产、被抄家、入狱	镜花水月一场空

游戏性：结局并非只在游戏结束时评定。你随时可能因重大决策失误而提前迎来结局。这创造出一种持续的紧张感和对每一个决策的珍视。

---

(十) 辅助玩法

1. 建筑经营
- 总号与各分号有建筑布局界面
- 扩建库房（增加存银容量）、加固金库（防盗）、增设账房（提升业务处理速度）
- 建筑风格体现晋商文化与城市特色

2. 收藏与鉴赏
- 经营过程中获得古玩字画、珍本账册（既是收藏也是隐性财富）
- 可设鉴赏室向来访贵客展示，提升声望

3. 方志编修
- 游戏中经历的重大事件、结识的重要人物、创下的纪录会被记录在一本动态生成的"票号方志"中
- 方志本身成为玩家回顾历程的情感载体

# 下篇：MVP 开发计划

> 下篇是面向第一版可落地的简化设计方案。所有设计决策以"能在个人/小团队规模下开发完成"为前提，同时保持核心命题"信义与利益平衡"不丢失。

## 〇、MVP 范围说明

### 什么是 MVP

MVP（最小可行产品）的目标不是"做一个小游戏"，而是**用最小的系统集合验证核心循环是否好玩**。如果 MVP 的核心循环成立，后续系统可以逐步叠加；如果不成立，说明设计基调需要调整，此时投入越少损失越小。

### MVP 取舍原则

| 原则 | 说明 |
|------|------|
| **核心命题不丢** | "信义 vs 利益"的每一次抉择都必须在 |
| **砍广度，保深度** | 宁可在 3 个系统上做深，不在 9 个系统上摊薄 |
| **文字为主，图形为辅** | MVP 以文字+数据面板呈现，不做复杂 UI 交互 |
| **数值模拟，不做 AI** | 分号用数值面板驱动，不做独立人格 AI |
| **预设代替自定义** | 密押、建筑等用预设方案，不给玩家编辑器 |

### MVP 包含 vs 暂缓

| 模块 | MVP 范围 | 远景版本 |
|------|---------|---------|
| 分号 | 1总号 + 2分号 | N个分号 + AI掌柜 |
| 人才 | 2层（学徒→账房） | 5层 + 师徒 + 派系 |
| 密押 | 3种预设方案 | 自定义编辑器 |
| 业务 | 汇兑 + 存银 + 简版放款 | 全部 + 官银竞标 |
| 小游戏 | 数值检定 | 验银/验票 mini-game |
| 官府 | 事件触发 | 完整官场系统 |
| 商帮 | 2~3个，好感度数值 | 5大商帮动态网 |
| 事件 | 5~8个核心剧情事件 | 30+事件 + 时代系统 |

## 一、MVP 核心经营循环

MVP 的核心循环由三个时间粒度组成：**晨间简报（每日）→ 季度决策（每季）→ 年终合账（每年）**。三者形成"信息获取 → 战略调整 → 结果检验"的完整闭环。

### 1. 晨间简报（每日 / 每回合起始）

**形式**：每天清晨，案头呈上当日必须处理的简报。以文字叙事 + 2~4 个选项的方式呈现。

**简报内容**（每次随机抽取 3~5 条）：

| 类型 | 示例 | 玩家决策 |
|------|------|---------|
| 银库日报 | "汉口分号库银较昨日减少八千两，准备金率降至 35%" | 标记关注 / 暂不处理 |
| 汇票审批 | "京师分号呈上大额汇票一张，面额六万两，密押校验通过" | 用印批准 / 驳回重验 |
| 情报简报 | "跑街来报：恰克图俄商皮毛到货量翻倍，急需汇兑渠道" | 标记商机 / 暂不跟进 |
| 人事呈报 | "学徒赵三已学满基础密押，账房李先生建议升为跑街" | 批准晋升 / 再考察半年 |
| 突发事件 | "张家口分号来函：当地谣言称我号存银不足，有小股存户排队取款" | 立即拨银 / 发公告辟谣 / 静观其变 |

**关键设计**：
- 每回合只能处理有限数量的简报（如 3 条），其余自动忽略或由系统代理。这制造"注意力稀缺"。
- 分号汇报内容经过**数值过滤**——系统根据掌柜能力值，可能隐瞒或淡化某些信息。但 MVP 中不模拟独立 AI 人格，而是在简报文本上叠加"可靠度"标记（如"本消息可信度：中"）。

---

### 2. 季度经营决策（每季度末）

**形式**：账房汇总本季经营数据，呈上决策清单。玩家在 4~6 个决策项中做出选择。

**决策项（简化版）**：

| 类别 | 决策项 | 选项范围 | 影响 |
|------|--------|---------|------|
| 资金 | 白银调拨 | 从总号向分号 A/B 调拨 X 万两 / 不调拨 | 影响各地准备金率 |
| 资金 | 放款额度 | 本季放款上限：保守(5万) / 适中(10万) / 积极(20万) | 利润 vs 坏账风险 |
| 定价 | 汇费调整 | 汉口号汇费：低(2%) / 中(3.5%) / 高(5%) | 竞争力 vs 利润率 |
| 定价 | 存银利率 | 存款利率：低(1%) / 中(2%) / 高(3%) | 存银量 vs 利差 |
| 人事 | 伙友调度 | 将某伙友从总号派往分号 / 留在总号 / 提拔 | 人才培养方向 |
| 巡查 | 分号巡查 | 派巡查去分号 A / 分号 B / 不巡查 | 发现问题的概率 |

**关键设计**：
- 决策效果有**1~2 个季度的延迟**。本季调拨的白银，下季才到；本季提拔的伙友，两个季度后才看到效果。
- 每个决策都可以看到**预估影响**（如"调拨 5 万两，预计汉口准备金率升至 45%"），但预估有±15% 的误差。

---

### 3. 年终合账（每年腊月）

**形式**：一年一度的核心大事件日。各分号账目汇总到总号，玩家进行年终审计和来年规划。

**流程**：

**第一步：对账**
- 系统呈现各分号年度账目与总号底账的比对结果
- 若出现"对不上"的情况，触发对账事件：展示差异金额和掌柜的解释
- 玩家选择：**亲自查账**（消耗"行动力"，但能查出真相）或 **采信解释**（省时但有风险）
- 取消远景版中的 mini-game，改为文字+数值判定的方式

**第二步：分红与留利**
- 年末利润分配，三项分配：
  - 东家分红：影响东家信任度
  - 票号留存：影响下一年可支配资本
  - 伙友花红：影响全员士气

**第三步：人事评定**
- 各分号掌柜/账房年度表现一览，玩家决定：
  - 续任 / 调离 / 辞退
  - 提拔新人填补空缺

**第四步：来年规划**
- 设定下一年目标（如"提升信誉至 X 级""新设 1 个代办处"）
- 目标达成与否影响年终评价

## 二、MVP 资源系统（简化版）

三大核心资源（白银、信誉、人脉）在 MVP 中保留，但结构大幅简化。

### 1. 白银 —— 单一总库制

MVP 中不区分"库银 / 在途银 / 存银"三层结构，改为**单一白银总量** + **拆分明细**：

| 字段 | 说明 |
|------|------|
| **总银两** | 票号持有白银总量（含总号 + 各分号） |
| **总号存银** | 平遥总号地窖中的可用白银 |
| **汉口库银** | 汉口分号库房存银 |
| **张家口库银** | 张家口分号库房存银 |
| **在途银** | 正在调拨途中、暂时不可用的白银 |

**关键规则（简化版）**：
- **准备金率** = 各地库银之和 ÷ 应付存款总额。低于 30% 触发信誉预警。
- **调拨成本**：从总号向分号调拨白银，消耗白银（运费 + 护卫费），延迟 1~2 个季度到账。
- **铜钱兑换**：暂时不做兑换比价波动，简化为 1 两白银 = 1000 文铜钱固定汇率。

**MVP 收支表**：

| 收入来源 | 支出项目 |
|---------|---------|
| 汇兑汇费（汇兑额 × 汇费率） | 伙友薪俸（每人每月固定） |
| 存银利差（放款利息 - 存款利息） | 分号运营费（每个分号每月固定） |
| 放款利息 | 调拨运费 + 护卫费 |
| | 兑付支出（客户提款） |

---

### 2. 信誉 —— 五级简化制

远景版中信誉有四个维度（兑付 / 密押 / 口碑 / 官印）。MVP 合并为**单一信誉值 + 五级标签**：

| 信誉等级 | 名称 | 影响 |
|---------|------|------|
| ★★★★★ | 金字招牌 | 存款利率最低可设 1%，存款自然流入，可承接大宗官银 |
| ★★★★ | 信誉卓著 | 存款利率最低可设 1.5%，商人主动上门汇兑 |
| ★★★ | 中规中矩 | 正常经营，无加成无惩罚 |
| ★★ | 风雨飘摇 | 存款外流加速，商人减少汇兑，需提高存款利率留客 |
| ★ | 信用破产 | 存款大量流失，几乎无人敢汇兑，进入死亡倒计时 |

**信誉变动规则**：
- 每次成功足额兑付：+1~3 点
- 每次拒兑或拖延：-10~20 点
- 谣言/负面事件：-5~15 点/季
- 信誉值从 0 到 100，对应五个等级区间
- 信誉下降速度是上升速度的 5 倍（"易毁难建"原则在数值上落地）

---

### 3. 人脉 —— 三类好感度

MVP 将人脉简化为三类**数值型好感度**：

| 人脉类型 | 数值范围 | 主要作用 | 消耗方式 |
|---------|---------|---------|---------|
| **官府好感** | 0~100 | 触发官银汇兑事件、获取政策风声 | 送礼、应酬消耗白银 |
| **商帮好感** | 0~100 | 大宗汇兑业务、危机时拆借支持 | 担保、让利、帮衬对方 |
| **东家信任** | 0~100 | 决定放权程度、能否追加资本 | 分红比例、经营稳健度 |

**人脉维护**：每季度自动消耗少量白银（如 500 两）作为"交际费"。若不维护，好感度每季自然衰减 1~3 点。重大事件可能瞬间大量增减（如帮某官员渡过难关 +30，拒绝帮某商帮 -20）。

## 三、MVP 组织机构（简化版）

### 1. 分号网络：1 总号 + 2 分号

MVP 中票号网络固定为三个节点：

| 节点 | 类型 | 初始库银 | 商业特点 |
|------|------|---------|---------|
| **平遥总号** | 总号 | 30 万两 | 决策中枢，银库总部，密押中心 |
| **汉口分号** | 一等分号 | 15 万两 | 长江中游商业枢纽，茶叶贸易旺盛，存银业务量大 |
| **张家口分号** | 二等分号 | 8 万两 | 对蒙俄贸易门户，汇兑需求大但季节波动剧烈 |

**分号状态面板（每个分号一张卡）**：

```
┌─────────────────────────────────┐
│  汉口分号                        │
│  掌柜：李守义  能力：75  忠诚：82 │
│  ─────────────────────────────  │
│  库银：12.5 万两                 │
│  准备金率：42%  ████████░░ 安全  │
│  本月业务量：汇兑 18 笔 / 存银 3 笔│
│  经营状况：正常运转               │
│  ─────────────────────────────  │
│  [调拨白银] [更换掌柜] [派巡查]   │
└─────────────────────────────────┘
```

**MVP 关键简化**：
- 分号**不设独立 AI**，掌柜仅有"能力值"和"忠诚度"两个数值
- 能力值决定分号经营效率（业务量 × 能力系数）
- 忠诚度决定汇报信息的准确度（忠诚度 100 = 完全准确，忠诚度 60 = 有 40% 概率隐瞒问题）
- 玩家可通过"派巡查"提升信息准确度（一次性消耗白银，揭示该分号当前真实状态）

---

### 2. 人力资源：2 层简化培养

MVP 将远景版的 5 层岗位压缩为 **2 层 + 玩家（大掌柜）**：

| 岗位 | 人数上限 | 培养周期 | 主要功能 |
|------|---------|---------|---------|
| **学徒** | 6 人 | 3~6 个月 | 基础劳动力，可分配至各分号做杂役 |
| **账房/跑街** | 4 人 | 由学徒晋升 | 放在总号或分号，提升该地业务处理能力 |
| **大掌柜（玩家）** | 1 人 | — | 全局决策 |

**伙友属性（简化版）**：

| 属性 | 说明 |
|------|------|
| **资质**（1~100） | 决定培养速度和能力上限 |
| **忠诚**（1~100，隐藏值，需观察推断） | 决定叛变概率和信息隐瞒概率 |
| **能力**（1~100，随历练增长） | 派驻分号时影响当地经营效率 |
| **天赋**（随机，约 20% 伙友拥有） | 特殊加成，如"铁算盘（对账效率+20%）""善交际（商帮好感获取+30%）" |

**培养流程**：
1. **招募**：三种途径——乡党举荐（免费，素质随机）、公开招考（消耗 500 两，可筛选）、商帮引介（消耗人脉好感）
2. **学徒期（3~6 个月）**：自动积累经验，玩家可安排"师傅"加速培养
3. **晋升决策**：学徒期满后，玩家决定——晋升为账房/跑街，或继续考察，或辞退
4. **派驻**：将账房/跑街派驻到分号，提升分号经营能力

> 远景版中的"分号掌柜"角色在 MVP 中取消。总号与两个分号的管理决策由玩家直接做出，不再通过独立的 AI 掌柜中转。这样可以大幅降低系统复杂度，同时保留"人事管理"的策略感。

---

### 3. 伙友风险事件（MVP 简版）

| 事件 | 触发条件 | 应对选项 |
|------|---------|---------|
| 学徒考核不合格 | 资质 < 30 的学徒期满 | 再给机会 / 辞退 |
| 账房做假账 | 忠诚 < 40 且经手大额账目 | 警告 / 辞退并报官 / 私下处理 |
| 伙友被挖角 | 能力 > 80 的伙友，竞争对手高薪挖人 | 加薪挽留 / 放手 |
| 账房泄密 | 忠诚极低 + 接触密押 | 更换密押 / 追查内奸 |

## 四、MVP 业务系统

### 1. 汇兑业务（核心）

MVP 中汇兑是自动运转的**被动收入来源**，玩家不参与每一笔汇票的开具和验票，而是通过**定价策略**和**密押方案选择**间接影响。

**运作方式**：
- 各分号每季度自动产生汇兑业务量，取决于：当地商业活跃度 × 票号信誉系数 × 汇费竞争力
- 汇兑收入 = 当季汇兑总额 × 汇费率（玩家设定）
- 玩家可在季度决策中调整各分号的汇费定价

**密押系统（MVP 预设方案）**：

MVP 取消自定义密押编辑器，改为 **3 种预设密押方案**，玩家选择并定期更换：

| 方案 | 防伪强度 | 学习成本（学徒掌握所需月数） | 开票效率 | 升级花费 |
|------|---------|--------------------------|---------|---------|
| **基础密押** | 60 | 2 个月 | 高（每笔耗时短） | 2000 两 |
| **标准密押** | 80 | 4 个月 | 中 | 5000 两 |
| **高级密押** | 95 | 6 个月 | 低（每笔耗时长） | 12000 两 |

**密押相关事件**：
- 密押泄露风险随"使用时间"递增——同一套密押使用超过 12 个月，泄露概率显著上升
- 当出现"假票兑付成功"事件时，玩家必须选择：局部更换（5000 两）/ 全面更换（15000 两）

---

### 2. 存银业务

**运作方式**：
- 各分号按玩家设定的存款利率自动吸纳存款
- 存款利率越高 → 存银越多 → 准备金率分母变大 → 利差空间被压缩
- 存银按一定比例转化为可放款的资金池

**玩家决策**：
- 每季度可调整各分号的存款利率
- 高信誉（★★★★以上）时，可在低利率下仍吸引存款
- 低信誉时，需要更高的利率才能留住存户

**存款结构（MVP 简化）**：

| 客群 | 特点 |
|------|------|
| 商人周转金 | 额度大、存期短（1~2 季）、对利率不敏感 |
| 官绅私蓄 | 额度大、存期长（2~4 年）、对信誉极敏感 |
| 百姓零存 | 额度小但稳定、基本不受利率影响 |

---

### 3. 放款业务（简化版）

MVP 中放款简化为**季度额度 + 事件触发**的模式：

- 玩家在季度决策中设定"本季放款上限"
- 系统根据上限自动匹配合适的借款对象（按风险从低到高）
- 利息收入 = 放款总额 × 平均利率

**特殊放款事件**（打断自动流程）：

| 事件 | 描述 | 决策 |
|------|------|------|
| 商帮求助 | "徽商李老板急需 8 万两周转三个月，以扬州盐引作抵" | 批准 / 拒绝（影响商帮好感） |
| 官府垫款 | "知府大人请求垫付 5 万两赈灾款，承诺来年税银到后偿还" | 垫付 / 婉拒（影响官府好感） |
| 熟人拆借 | "老主顾张掌柜想借 3 万两，无抵押，凭多年交情" | 同意 / 要求担保人 |
| 高风险高利 | "陌生商人出 20% 年利借 10 万两，背景不详" | 拒绝 / 冒险放款 |

**坏账处理**：借款到期未还时，触发追讨事件（选择：协商延期 / 执行抵押 / 找担保人 / 放弃），参见远景版设计。

---

### 4. 官银汇兑（MVP 中改为事件触发）

MVP 不设完整的官银竞标系统和官场经营，改为**随机触发的事件**：

- 某季度可能触发"官员上门"事件——某省衙门拟将一笔官银汇兑交给你处理
- 事件附带条件（如"需要在账目上略作修饰""需要私下返点 X 千两"）
- 玩家接受/拒绝，立即产生结果（白银 + 信誉 + 官府好感变动）
- 之后可能触发后续——"该官员被查办，你的票号被牵连调查"

## 五、MVP 事件与剧情设计

### 1. 核心剧情事件（固定剧本）

MVP 首发保留以下 5 个精心设计的剧情事件：

**事件一：密押之夜**
- **触发**：随机，游戏进行到第 2~5 年之间
- **情节**：总号密押总册不翼而飞，疑有内鬼
- **分支**：
  - 全面更换密押（花费 15000 两，安全但损失大）
  - 局部更换 + 追查内奸（花费 8000 两 + 随机结果）
  - 按兵不动，静观其变（不花钱，但可能有假票出现）
- **设计目的**：让玩家第一次直面"防伪安全 vs 成本"的抉择

**事件二：东家的考验**
- **触发**：东家信任度降到 40 以下
- **情节**：东家突然要求亲自查账
- **分支**：
  - 坦荡配合（账目干净则信任回升；账目有问题则被质询）
  - 暗中运作拖延（消耗人脉 + 白银）
  - 主动请辞（赌东家挽留，极高风险）
- **设计目的**：提醒玩家注意东家关系，也是经营成果的一次"审计"

**事件三：商帮危机**
- **触发**：徽商好感度 ≥ 60 时随机触发
- **情节**：徽商遭遇资金链危机，向你紧急求助
- **分支**：倾力相助 / 量力而行 / 推辞 — 不同选择深度影响商帮好感及后续业务
- **设计目的**：展现晋商"利在长远"的核心价值观

**事件四：巡抚的橄榄枝**
- **触发**：官府好感度 ≥ 50
- **情节**：新任巡抚欲将一省地丁钱粮交给你汇兑，但要求账目"略作修饰"
- **分支**：答应 / 婉拒 / 假意答应但留后路
- **设计目的**：核心命题"庙堂与江湖的平衡"的直接呈现

**事件五：挤兑风暴**
- **触发**：准备金率 < 30% 或信誉 < ★★★
- **情节**：某分号出现存户排队提款
- **分支**：
  - 从总号紧急调银（花费大）
  - 向同业拆借（消耗人脉）
  - 暂停兑付（信誉断崖式下跌）

---

### 2. 分号危机事件

每个分号独立随机触发以下危机，频率约每 2~3 年一次：

| 危机 | 触发条件 | 应对选项 | 设计意图 |
|------|---------|---------|---------|
| 在途被劫 | 有在途银时随机 | 报官 / 雇镖局追讨 / 自担损失 | 让"在途银"概念产生实际紧张感 |
| 水火天灾 | 纯随机 | 拨款重建 / 暂迁业务 | 测试应急储备 |
| 同业价格战 | 竞争对手动态 | 跟进降价 / 差异化竞争 / 协商 | 引入竞争维度的策略选择 |

---

### 3. 随机小事件（填充日常）

在晨间简报中穿插以下小型随机事件，提供叙事色彩和资源微调：

- 某跑街捡到客商遗落银票（选择：归还 / 私吞 — 影响品行）
- 平遥庙会，总号需出份子钱（消耗少量白银，提升乡党好感）
- 老伙友病倒，需寻医问药（消耗白银，增加伙友忠诚）
- 竞争对手在隔壁街开新号（增加经营压力，可选应对）
- 洋人商船延误导致某地茶价波动（影响特定分号业务量）

### 3. 核心游戏循环（伪代码）

```
function gameTick(season) {
  // 1. 季度自动模拟
  for (branch of branches) {
    branch.revenue = simulateRemittance(branch, pricing, reputation);
    branch.deposits = simulateDeposits(branch, pricing.depositRate, reputation);
    branch.expenses = calculateExpenses(branch, staff);
  }

  // 2. 随机事件判定
  const events = rollForEvents(gameState);

  // 3. 经营数据刷新
  updateSilverState(gameState);
  updateReputation(gameState);

  // 4. 将玩家决策推入待处理队列
  if (season.end) pushQuarterlyDecisions();
  if (year.end) pushAnnualSettlement();
}
```

### 4. MVP 不做的技术点（明确排除）

- ❌ 不做验银/验票 mini-game 的图形交互
- ❌ 不做官银竞标的对手 AI
- ❌ 不做多人/联网功能
- ❌ 不做 3D 或复杂动画
- ❌ 不做移动端适配（先聚焦桌面 Web）
