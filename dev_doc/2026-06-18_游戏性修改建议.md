# 《晋·信》游戏性修改建议

> **日期**：2026年6月18日  
> **基于**：[2026-06-18_游戏性审查报告.md](2026-06-18_游戏性审查报告.md)  
> **原则**：每项建议标注优先级（P0/P1/P2/P3）、实现复杂度（低/中/高）、预期效果

---

## 一、核心循环：打破重复，引入节奏变化

### 建议 1：季度决策瘦身 — 5+2 → 3+1

- **优先级**：P0
- **复杂度**：低
- **问题**：每季 6~7 项决策，5 年 120+ 次选择，70% 选项效果无感知
- **方案**：

```
当前：5 项必出（调拨/汇费/利率/放款/巡查）+ 1~2 项动态 = 6~7 项/季
改为：3 项必出（银两运营/费率定价/人事调度）+ 1 项动态 = 4 项/季
```

具体合并方式：
| 原决策 | 合并后 |
|--------|--------|
| 白银调拨 | **银两运营**（合并调拨 + 放款额度 + 巡查为 3 选项） |
| 汇费调整 | **费率定价**（合并汇费 + 存贷利率为联动选项） |
| 存贷利率 | ↑ 并入费率定价 |
| 放款额度 | ↑ 并入银两运营 |
| 分号巡查 | ↑ 并入银两运营 |
| 动态 1~2 项 | 保持 1 项动态 |

- **效果**：每季选择量 **6~7 → 4**，每年 24~28 → **16**，5 年 120~140 → **80**。每个决策的信息密度和重要性提升。
- **文件**：`src/engine/decisions.ts`（重构 `generateQuarterlyDecisions`）

### 建议 2：晨间简报 → 季度摘要（可跳过）

- **优先级**：P1
- **复杂度**：中
- **问题**：3~5 条简报是纯信息，第 3 季起玩家无脑跳过
- **方案**：

```
当前：逐条弹出 3~5 张卡片，必须点"知晓"
改为：一页季度摘要面板，顶部 1 句核心摘要 + 可展开详情
```

**新 UI 结构**：

```
┌─────────────────────────────────────────┐
│  嘉庆十三年 · 春 · 季度摘要              │
│                                         │
│  💰 汉口汇兑 97,600 两，净利 +814 两     │
│  📊 准备金率 22%，偏低                   │
│  👤 密押已用 6 月，9 月后需更换           │
│                                         │
│  [查看详情 ▸]     [进入季度决策 →]       │
└─────────────────────────────────────────┘
```

- **效果**：简报从"强制阅读障碍"变为"一览即过的参考信息"。新手可以展开详情，老手可以直接进入决策。
- **文件**：`src/components/morning/MorningPanel.tsx`（重构），`src/engine/morning.ts`（简化输出）

### 建议 3：引入季度节奏变化 — "忙季"机制

- **优先级**：P1
- **复杂度**：中
- **问题**：每季节奏完全一致，没有起伏
- **方案**：

| 季节 | 类型 | 特殊规则 |
|------|------|---------|
| 春 (S1) | 常规季 | 正常决策，放款需求↑ |
| 夏 (S2) | 风险季 | 汛期/暑热，突发事件概率↑至 50%，在途风险↑ |
| 秋 (S3) | 旺季 | 汇兑量 ×1.5，决策项 +1（临时雇工等旺季选项） |
| 冬 (S4) | 结算季 | 直接进入年终合账，无常规季度决策 |

- **效果**：四季有不同的期待感和策略重心。冬季跳过季度决策，直接进入年终合账，形成自然的"一年结束"仪式感。
- **文件**：`src/engine/simulation.ts`（季节倍率），`src/engine/decisions.ts`（季节决策差异）

## 二、建设维度：让玩家感觉"在建造什么"

### 建议 4：分号扩张系统

- **优先级**：P0
- **复杂度**：高
- **问题**：经营 5 年永远是 2 个分号，没有扩张感
- **方案**：

**新增第三分号**：1812 年（第 4 年）触发事件"西安设号之议"

| 条件 | 银两要求 | 效果 |
|------|---------|------|
| 在西安设分号 | 总号库银 ≥ 80,000 + 信誉 ≥ 50 | 新增 tier 2 分号，基础业务量 40k/季 |
| 暂缓扩张 | — | 保持 2 分号，商帮好感 -5 |

设号后：
- 季度决策中新增"西安"相关选项
- 需要派驻伙友/招聘新掌柜
- 西安为三等分号（tier 3, 倍率 0.7），可通过投资升级至二等
- 新增"西安→兰州"潜在商路

**效果**：玩家在中期有一个"里程碑"。1812 年不再只是"又一年"。

**文件**：`src/types/index.ts`（Branch 支持 tier 3），`src/data/branches.ts`（西安分号初始数据），`src/engine/simulation.ts`（三分号模拟），`src/engine/events.ts`（设号事件），`src/components/branch/BranchPanel.tsx`（三分号展示）

---

### 建议 5：分号升级系统

- **优先级**：P2
- **复杂度**：中
- **问题**：分号等级（tier 1/2）是固定的，不能升级
- **方案**：

年终合账 Step 4"来年规划"中，若选择"开疆拓土"：
- 选择一个分号投入 30,000 两升级（tier 2→1 或 tier 3→2）
- 升级需 2 季完成
- 完成后：等级倍率提升 + 业务量基础值 +20%

- **文件**：`src/engine/annual.ts`，`src/components/annual/AnnualPanel.tsx`

## 三、目标系统：让每一年都有意义

### 建议 6：里程碑成就系统

- **优先级**：P1
- **复杂度**：低
- **问题**：5 年经营没有任何中间里程碑
- **方案**：触发式成就弹窗

| 里程碑 | 触发条件 | 效果 |
|--------|---------|------|
| 🏆 初露锋芒 | 首季盈利 | 士气提振（全员 loyalty+2） |
| 🏆 万两之槛 | 总银首次突破 100 万 | 商帮好感 +10 |
| 🏆 金字招牌 | 信誉首次达到 85 | 解锁"金字招牌"礼遇（存息自动-0.2%） |
| 🏆 桃李满门 | 首个学徒出师 | 免费获得 1 名随机学徒 |
| 🏆 官商之交 | 官府好感首次达到 70 | 解锁稀有机会"官银汇兑大单" |
| 🏆 三年无事 | 连续 3 年无潜逃/危机 | 全员 loyalty+5 |

- **实现**：在 `applySimulation` 或 QuarterlyPanel 中检测成就条件，触发 Toast 弹窗和日志记录
- **文件**：新建 `src/engine/achievements.ts`，`src/components/common/Toast.tsx`（已存在）

---

### 建议 7：年度规划赋予实际效果

- **优先级**：P1
- **复杂度**：低
- **问题**：年终"开疆拓土/固本培元/信誉至上"是空壳选项
- **方案**：

| 选项 | 下一年效果 |
|------|-----------|
| 开疆拓土 | 下一年汇兑业务量 +15%，运营支出 +20%，巡查可选 0 两(节约) |
| 固本培元 | 下一年运营支出 -10%，放款坏账率 -50%，但汇兑增长仅 +5% |
| 信誉至上 | 下一年信誉变动 +2/季（额外加成），但汇费不可上调 |

- **实现**：GameState 新增 `annualDirective: string` 字段，simulation 中根据该字段调整倍率
- **文件**：`src/types/index.ts`，`src/engine/simulation.ts`，`src/engine/annual.ts`

## 四、人脉系统：从被动数字变为主动资源

### 建议 8：人脉可主动"消费"

- **优先级**：P1
- **复杂度**：中
- **问题**：人脉三个数字只能被动涨跌，玩家从不主动查看
- **方案**：在季度决策中增加 1 项"人脉运作"动态决策（替换为有意义的消耗选项）

| 运作方式 | 消耗 | 收益 | 风险 |
|----------|------|------|------|
| 疏通官府 | government -15 | 下季免巡检（默认已查，省 2000 两） | — |
| 商会游说 | merchantGuild -20 | 汇费可临时上调 +2%（本季汇兑收入 +15%） | — |
| 向东家进言 | owner -15 | 放款上限临时 ×2（本季放款收入翻倍） | 若本季亏损，owner 再 -10 |

冷却机制：每人脉维度每 4 季只能运作 1 次。

- **效果**：人脉从"三个僵尸数字"变成"需要经营的战略资源"。玩家会主动关注人脉数值。
- **文件**：`src/engine/decisions.ts`（新增 `generateConnectionAction`）

## 五、叙事整合：让故事影响经营，经营影响故事

### 建议 9：事件后果追踪与回调

- **优先级**：P2
- **复杂度**：中
- **问题**：事件处理即消失，没有后续影响
- **方案**：事件选择后，在后续季度简报中出现"后续"条目

| 事件 | 选择 | 2~3 季后的简报回调 |
|------|------|-------------------|
| 官银汇兑 | 接了 | "去年承运的官银悉数到账，太原府来函致谢"→ gov +3 |
| 汉口洪灾 | 赈灾 | "汉口商会在碑上刻了本号名字"→ 汉口业务量永久 +3% |
| 伙友跳槽 | 放人 | "听说他在侯家号里不受重用，常在酒楼叹气" |
| 密押泄露 | 暗查 | "暗查至今无果，但昨夜有人塞了封信到号门下..."→ 触发后续二级事件 |

- **实现**：GameState 新增 `eventConsequences: Array<{originEventId, season, description}>`，简报生成时遍历
- **文件**：`src/types/index.ts`，`src/engine/morning.ts`，`src/engine/events.ts`

---

### 建议 10：故事链分支 — 选择塑造后续剧情

- **优先级**：P3
- **复杂度**：高
- **问题**：5 个主线事件独立，无因果链
- **方案**：

```
官银汇兑(接)
  └─→ 官府线后续：朝廷嘉奖 → 军饷专汇权 → 卷入贪墨案(终极选择)
官银汇兑(拒)  
  └─→ 江湖线后续：商帮会首赏识 → 民间资本涌入 → 挤兑保卫战(终极选择)
```

- **无需 10 个独立事件，只需 6 个节点 + 分支引用**。每个节点 1 个后续事件（共 4~6 个额外事件）。
- **文件**：`src/data/events.ts`（新增 4~6 个分支事件 + 条件引用）

## 六、终局体验：从突兀到高潮

### 建议 11：终局预警系统

- **优先级**：P1
- **复杂度**：低
- **问题**：结局在例行结算中突然降临
- **方案**：

当接近结局条件时，在晨间简报/季度摘要中显示预警：

| 距离 | 触发条件 | 预警文案 |
|------|---------|---------|
| 金玉满堂·近 | 总银 ≥ 100 万 且 年 ≥ 1814 | "库银已逼近金玉满堂之数...再拼一把？" |
| 守成之主·近 | 年 = 1812 | "五年之期将至。这一年，不求有功，但求无过。" |
| 急流勇退·可 | 信誉 ≥ 80 且 总银 ≥ 80 万 且 年 ≥ 1817 | "功成名就，此时退隐正是时候。" |
| 惨淡收场·危 | 总银 < 10,000 | "库银见底——再不逆转，票号将无力回天。" |

- **效果**：玩家知道"终点近了"，最后几季有冲刺的紧张感。
- **文件**：`src/engine/morning.ts`（新增终局预警简报模板）

---

- **优先级**：P1
- **复杂度**：中
- **问题**：GameOverPanel 只有几行统计，没有回顾感
- **方案**：在现有 GameOverPanel 基础上增加 3 个板块

**板块 1：经营走势图**（纯 CSS 柱状图）
```
银两变化（每季）：
  1808春 ████████ 77.0万
  1808夏 ████████▌ 77.8万
  1808秋 █████████ 78.6万
  ...
```

**板块 2：关键抉择回顾**（从日志中提取事件抉择）
```
嘉庆十三年秋 · 你选择了"全力接单"应对汇兑潮涌
嘉庆十四年春 · 你拒绝了东家的索银要求
嘉庆十五年冬 · 你在洪灾中开仓赈灾
```

**板块 3：伙友去向**
```
陈守义 — 始终追随，总号账房荣休
王有财 — 嘉庆十五年出师，现任汉口分号账房

```

- **文件**：`src/components/gameover/GameOverPanel.tsx`（新增 3 个复盘板块），新增 `src/engine/retrospective.ts`（生成复盘数据）

## 七、复玩性：给重开一个理由

### 建议 13：开局阵营选择

- **优先级**：P3
- **复杂度**：中
- **问题**：每次重开体验完全一致
- **方案**：在难度选择后增加"出身"选择

| 出身 | 初始优势 | 初始劣势 | 推荐玩法 |
|------|---------|---------|---------|
| 商贾世家 | 商帮好感 70，东家信任 80 | 官府好感 20 | 民间路线 |
| 官绅门第 | 官府好感 70，初始库银 +5 万 | 商帮好感 25 | 依附庙堂 |
| 白手起家 | 全员 loyalty+10，运营成本 -20% | 初始信誉 45，人脉全 30 | 草根逆袭 |

- **效果**：3 种出身 × 2 种难度 = **6 种不同开局**。不同出身引导不同策略路线。
- **文件**：`src/data/constants.ts`（3 套初始参数），`src/components/auth/DifficultySelect.tsx`（出身选择）

---

### 建议 14：随机事件池差异化

- **优先级**：P2
- **复杂度**：低
- **问题**：每局随机事件从同一个池子里抽
- **方案**：将 13 个随机事件分为 3 组，每局随机启用 2 组（约 9 个事件）。未启用的事件在本局永不触发。

- **效果**：每局事件组合不同，不同局之间体验有差异。
