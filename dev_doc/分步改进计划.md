## 阶段十二：人事系统生效

### 目标
让人事评定的三个决策（续任/调离/辞退）产生实际游戏影响；学徒自动成长出师；伙友能力接入模拟引擎。

### 12.1 新增 state actions

**文件**：`hooks/useGameState.ts`

| Action | 逻辑 | 预期 |
|--------|------|------|
| `UPDATE_STAFF { id, changes }` | 按 id 更新 Staff 字段 | `loyalty`、`stationedAt` 可变更 |
| `REMOVE_STAFF { id }` | 从 `staff[]` 过滤移除 | 被辞退伙友不再出现在名册 |
| `PROMOTE_APPRENTICE { id }` | `role→clerk`、`ability = talent×0.9±10` | 学徒出师为正式伙友 |

**依赖**：无。（3 个 action → ~25 行）

### 12.2 人事评定接入实际效果

**文件**：`components/annual/AnnualPanel.tsx` → `handleStaff()`

```
当前：
  ADD_LOG + toast + setStep(4)

改为：
  遍历 decisions：
    续任 → UPDATE_STAFF loyalty+5
    调离 → stationedAt 轮换（headquarters↔hankou↔zhangjiakou），loyalty-5
    辞退 → REMOVE_STAFF + UPDATE_REPUTATION -2 + 全员 loyalty-3
  ADD_LOG + toast + setStep(4)
```

**依赖**：12.1。（~20 行）

### 12.3 学徒季度成长

**文件**：`engine/simulation.ts` → `applySimulation()`

在每季模拟结束时遍历 `staff[]`：`monthsTrained += 3`。若 `role === 'apprentice' && monthsTrained >= 36`，自动升级为 `clerk`，日志记录 `「{name} 历经三年磨练，今日正式出师。」`

**依赖**：12.1。（~12 行）

### 12.4 驻场伙友能力影响分号业务量

**文件**：`engine/simulation.ts` → `calcBranchRemittanceVolume()`

```
当前：volume = 基础量 × 信誉倍率 × (掌柜能力-50)/100 × 状态修正 × 等级修正

增加：驻场伙友修正
  驻场clerks = staff.filter(s => s.stationedAt === branchId)
  伙友均能力 = avg(驻场clerks.map(s => s.ability)) or 50（无人时默认50）
  staffModifier = 0.7 × 掌柜修正 + 0.3 × (伙友均能力/50)
  volume = 基础量 × 信誉倍率 × staffModifier × 状态修正 × 等级修正
```

**依赖**：无。（~8 行，替换现有单行公式）

### 预期结果

| 场景 | 效果 |
|------|------|
| 辞退低能伙友 → 派驻高能者 | 分号业务量 +10~30%，季收入提升 |
| 长期续任 → loyalty 累积 | 危机事件中不会叛逃 |
| 学徒 3 年不辞退 | 免费获得新 clerk，节约招聘成本 |
| 错误调离 → loyalty 掉到 <30 | 季度风险事件概率增加 |

## 阶段十四：伙友事件与薪资

### 目标
伙友忠诚→风险→经济产出的完整闭环；薪资自动扣除。

### 14.1 薪资扣减

**文件**：`engine/simulation.ts` → `calcOperatingExpenses()`

每季按 `staff[]` 中每人的 `role` 扣月薪×3，辞退后支出自动减少。

**依赖**：已有 `MONTHLY_SALARY` 常量。（~4 行）

### 14.2 忠诚危机事件

**文件**：`engine/quarterlyEvents.ts` 或 `engine/events.ts`

条件：任一伙友 `loyalty < 30` 时，每季 10% 概率触发「伙友携款潜逃」事件：该伙友所在分号库银 × 15% 被盗，伙友从 staff[] 移除。

**依赖**：12.1。（~15 行）

### 14.3 高能力伙友额外收入

条件：驻扎某分号的伙友 `ability > 80 && loyalty > 70`，每季额外为该分号贡献汇兑业务量 +10%。

**依赖**：无。（~5 行，在 simulation.ts 中并入）
