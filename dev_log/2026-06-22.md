**Web Audio API 程序化生成**（零外部音频文件、凭版权）：

- **五声音阶**（宫商角徵羽，12 个音高）→ 三角波模拟古筝拨弦泛音

## T4 — 后端用户认证系统

| 属性 | 值 |
|------|-----|
| **新增文件** | `edge-functions/api/auth/[[default]].js` |
| **修改文件** | `src/utils/accountManager.ts`、`src/hooks/useAuth.tsx`、`src/components/auth/LoginPage.tsx` |
| **状态** | ✅ 完成 |

### 架构

```
前端 fetch(/api/auth/*)  →  Edge Function (V8)  →  KV Storage (jinxin_kv)
     POST /register               PBKDF2-SHA256          user:{name}
     POST /login                  10万次迭代               session:{token}
```

### 安全设计

| 环节 | 方案 |
|------|------|
| 密码哈希 | PBKDF2-SHA256，16 字节随机盐，服务端 pepper |
| 传输 | HTTPS（EdgeOne 默认） |
| Session | 32 字节随机 token，KV 存储，1 小时过期 |
| KV 配置 | 命名空间 `jinxin_store`，变量名 `jinxin_kv`（控制台手动绑定） |

### 遇到的问题 & 解决

| 问题 | 原因 | 解决 |
|------|------|------|
| 前端报"无法连接云端服务" | V8 Edge Functions **不支持** `Response.json()` | 改为 `new Response(JSON.stringify(...))` |
| API 路由 404 | `api/auth.js` 仅匹配 `/api/auth`，不匹配 `/api/auth/register` | 改为 `api/auth/[[default]].js` catch-all |

## 今日关键决策

1. **KV Storage 而非 Cloud Functions**：Edge Functions + KV 比 Cloud Functions + 数据库更轻量，无需额外配置 MySQL/PostgreSQL
2. **程序化音乐而非 MP3 文件**：Web Audio API 免版权、离线可用、6KB 代码即完整音乐系统
3. **纯云端不走本地降级**：用户明确要求去掉 localStorage fallback，账号安全由后端全权负责
4. **V8 Runtime 踩坑**：Edge Functions 不支持 `Response.json()`、不支持 `process.env`、不支持 npm 包，需严格遵循 Web Standard API
