## 一、背景故事

### 楔子：天下变局中的一纸契约

**嘉庆十二年秋**，白莲教之乱的余烬在川楚深山里化作最后一缕青烟。紫禁城内，嘉庆帝对着空虚的国库长叹；而千里之外的**山西平遥**，南大街的青石板路上，驼铃声与算盘声交织成另一曲帝国的脉搏。

大清帝国由盛转衰，一个横跨欧亚大陆的贸易网络正以山西商人的脚步为针、以白银为线，悄然缝合着从武夷山茶田到恰克图皮货集市的天堑鸿沟。这场无声变革的心脏，就在你脚下的青砖灰瓦之间。

### 你的身份：票号大掌柜

日升昌、蔚泰厚等大号的红漆招牌在日光下晃得人眼晕，而你执掌的是一家中型票号。东家厌倦了官场倾轧，将万贯家财全权委托于你——**你是大掌柜，是这艘金融巨舟的实际掌舵人**。

### 你卖的是一张纸，守的是一口气

票号不卖米面绸缎，它卖的是一张薄薄的**汇票**。这张纸之所以能让商人从京师提走汉口分号的万两白银，凭的完全是票号招牌的信誉。晋商经营数百年，立下三条铁律：

| 铁律 | 含义 |
|------|------|
| **信为命脉** | 宁可赔光存银，不可让任何一张真票无法兑付。失信一次，便是灭门之灾 |
| **利在长远** | 不与相熟商帮做一锤子买卖。晋商的生意，是做给子孙后代的 |
| **庙堂与江湖** | 票号离不开官府，但绝不能沦为附庸。与巡抚过密，他倒台时你万劫不复 |

### 暗流之下的决断

此刻，你坐在昏暗的账房内，一盏油灯照亮了几份摊开的密函：

- **汉口分号急报**：茶市受累，银根紧缩，急需总号拨银救急
- **张家口密函**：俄商对晋商汇票信任日增，但对方暗中窥探密押底牌
- **平遥县衙登门**：新任巡抚欲将地丁钱粮交由你号汇兑，但需你在账簿上「略作修饰」

每一次落笔，都是在**信义与利益、规矩与人情、商道与官道**之间寻找细若游丝的平衡。

**你的目标绝非偏安一隅**——你要让票号翻过太行山、渡过长江、跨过戈壁，在乱世里筑起一座不倒的金融堡垒。当未来的史书翻开这一页时，那块由你亲手题写的金字招牌，将比任何白银都更加闪耀。

## 二、玩法介绍

### 宏观概览

《晋·信》是一款**以季为节奏、以年为轮回**的文字策略游戏。玩家从 1808 年春季开始，每季经历「晨间简报 → 季度决策 → 经济模拟」的完整循环，每年冬季结束后进入「年终合账」四步流程。经营目标是**在银两不断流、信誉不崩塌的前提下，达成特定胜利结局**。

### 核心资源体系

游戏经济系统建立在 **三大核心资源** 之上，彼此独立又深度联动：

| 资源 | 定位 | 核心机制 |
|------|------|----------|
| **白银** | 流通的血液 | 总号 + 汉口/张家口两分号的库银体系。准备金率、异地调拨成本、存贷利息差构成收支主干 |
| **信誉** | 票号的灵魂 | 五级评价体系（金字招牌 → 信用破产）。**易毁难建**：百年积累可毁于一夕拒兑 |
| **人脉** | 看不见的网 | 官府好感、商帮交情、东家信任三维并行。人脉是双刃剑——攀附官员获益，他倒台你遭殃 |

### 游戏循环

```
┌──────────────────────────────────────────────────┐
│                                                  │
│   季度摘要         季度决策        经济模拟      │
│  (一览即过)  →  (3+1 项决策) → (1.2s 算盘声)    │
│       ↑                              │           │
│       │                              ↓           │
│       └──── 来年规划 ← 人事评定 ← 分红方案       │
│                   ←──── 年终合账 ────┘           │
│                  （每年 Q4 结束后触发）           │
│                                                  │
└──────────────────────────────────────────────────┘
```

### 阶段系统

游戏通过 **6 个阶段（Phase）** 驱动流程：

| 阶段 | 说明 |
|------|------|
| `morning` | **季度摘要**：单页面板一览库银/准备金/信誉/密押关键指标，可选展开详细简报，一键进入决策 |
| `quarterly` | **季度决策**：3 项必出（银两运营/费率定价/伙友调度）+ 1 项动态，银两运营支持多选 |
| `simulating` | **模拟中**：算盘声 1.2 秒动画，后台计算汇兑收入/利息支出/运营成本/随机事件 |
| `event` | **事件剧情**：5 核心剧本 + 4 危机自救 + 14 季度突发事件，每事件 2~3 选 1，效果实时应用 |
| `annual` | **年终合账**：四步流程——对账审核、分红留利（三种方案含来年规划效果）、人事评定 |
| `game_over` | **终局结算**：6 种结局差异性展示，含预警提示 |

### 特色系统

| 系统 | 说明 |
|------|------|
| **分号网络** | 汉口一等分号（九省通衢）+ 张家口二等分号（口外据点），各有掌柜能力/忠诚/经营状态 |
| **伙友养成** | 初始 3 学徒 + 5 伙友。学徒季度成长（36 月出师），能力>80+忠诚>70 给分号额外加成 |
| **密押防伪** | 基础字码 → 标准双码 → 三码联动，逐级升级，密押泄漏信誉断崖 |
| **事件剧情** | 5 主线 + 4 危机自救 + 14 季度突发事件 + 10 角色池招聘，条件触发，多分支后果 |
| **突发事件** | 进入季度决策前 ~30% 概率触发（行情/商路/人事/机遇/招聘），信誉低时概率提升至 55% |
| **忠诚危机** | 伙友忠诚<30 时，每季 10% 概率携款潜逃，所在分号库银 -15% |
| **难度选择** | 普通模式（初始 77 万两 / 信誉 60）vs 困难模式（31 万两 / 信誉 40） |
| **里程碑成就** | 6 个阶段性成就（初露锋芒/万两之槛/金字招牌/桃李满门/官商之交/三年无事），触发时弹窗提示 |
| **年度规划** | 年终选择来年方向（开疆拓土/固本培元/信誉至上），下一年享有对应的经营加成 |
| **终局预警** | 濒临结局条件时，晨间自动推送醒目预警（如库银见底/金玉满堂临近/五年之期将至） |
| **HQ 枯竭防护** | 总号库银不足时，调拨/巡查选项自动置灰，晨间弹出紧急警告 |

### 结局系统（6 种）

| 结局 | 类型 | 达成条件 |
|------|------|----------|
| 🏆 **金玉满堂** | 胜利 | 总库银 ≥ 120 万两，且年份 ≥ 1815（经营 ≥ 7 年） |
| 🛡️ **守成之主** | 胜利 | 年份 ≥ 1813（坚持经营 5 年不倒） |
| 🎋 **急流勇退** | 特殊 | 信誉 ≥ 85，且年份 ≥ 1815，年终选择主动退隐 |
| 🚪 **功成身退** | 特殊 | 随时点击「结束游戏」按钮，即时结算 |
| 💸 **惨淡收场** | 失败 | 总库银 ≤ 0 两 |
| 📛 **晚节不保** | 失败 | 信誉 ≤ 0 |

## 三、技术栈

### 核心框架

| 层面 | 选型 | 版本 |
|------|------|------|
| UI 框架 | **React** | 18.3 |
| 类型系统 | **TypeScript** | 5.5 |
| 构建工具 | **Vite** | 5.4 |
| 样式方案 | **原生 CSS**（CSS Variables + 宣纸卷轴风格） | — |

### 状态管理与架构

| 组件 | 说明 |
|------|------|
| `useReducer` + `Context` | 全局 GameState 管理，28 种 Action 类型覆盖全部状态操作 |
| 引擎层纯函数 | `engine/` 目录下全部为无副作用纯函数（模拟/信誉/事件/年终） |
| localStorage | 多用户存档隔离（`jinxin_save_{username}`），含版本校验 |
| `@` 路径别名 | `@/` → `src/`，Vite resolve alias |

### 开发与质量

| 工具 | 用途 |
|------|------|
| `@vitejs/plugin-react` | JSX 自动转换 |
| `tsc --noEmit` | 每次提交前类型检查，保持零错误 |
| CC0 / 开源字体 | 阿里妈妈数黑体（标题）+ 抖音美好体（正文）+ Montserrat（数字） |
| 响应式 CSS | 宣纸卷轴古风主题，最小宽度 960px，适配桌面/平板 |

### 部署

| 平台 | 说明 |
|------|------|
| **EdgeOne Pages** | 静态站点 + Edge Functions + KV Storage 全栈托管 |
| **线上地址** | [jinxin-twlhpwzz.edgeone.cool](https://jinxin-twlhpwzz.edgeone.cool) |
| **任意静态服务器** | `dist/` 目录直接部署 |

### 后端架构

| 组件 | 说明 |
|------|------|
| **Edge Functions (V8)** | `edge-functions/api/auth/[[default]].js` — 用户注册/登录 API |
| **KV Storage** | `jinxin_kv` — 用户密码存储（PBKDF2-SHA256，10 万次迭代 + 随机盐） |
| **Session** | 32 字节随机 token，1 小时过期，KV 存储 |

### 安全体系（IOA）

| 模块 | 实现 |
|------|------|
| **数据交互校验** | 存档 16 位哈希签名验证 + 时间戳防回溯 |
| **玩家身份鉴权** | 云端 PBKDF2 密码哈希 + 服务端 pepper |
| **Agent 行为安全** | 100ms 操作间隔检测 + 单季频率上限 + 1h 自动锁定 |
| **异常行为识别** | 单季银两 >5 万/信誉 >20 告警 + 安全日志分级 |

### 背景音乐

**Web Audio API 程序化生成**（零外部文件、免版权）：
- 五声音阶（宫商角徵羽）+ 三角波模拟古筝拨弦泛音
- 4 种主题（经营/菜单/危机/胜利），自动切换
- 页面失焦自动暂停 + Sidebar 音量控制 + 静音开关

## 开发规范

- **日志记录**：每天开发结束后在 `dev_log/` 下以 `YYYY-MM-DD.md` 格式记录开发日志
- **详细设计**：见 `dev_doc/开发文档.md`（上篇为远景愿景，下篇为 MVP 开发计划）
- **类型安全**：所有新代码必须通过 `tsc --noEmit` 零错误检查
- **引擎纯函数**：`engine/` 目录下禁止引入 React，保持纯函数利于测试和复用
- **常量集中**：文案、颜色、数值参数集中在文件顶部或 `data/constants.ts`，便于调参
